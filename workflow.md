Detailed Behavior Flow for Each Role
Here is a clear and detailed user behavior flow for all 4 roles in your project:

1. Customer Flow

Registration / Login
Customer opens the website → Registers or Logs in (Phone/Email + OTP or Google)
Enters delivery address (or selects saved address)

Home Screen
Sees nearby available shops based on location
Can search products or browse categories

Browsing & Ordering
Searches or browses products
Views product details (price, stock availability, shop name)
Adds items to cart
Proceeds to checkout

Placing Order
Reviews cart and total amount
Places order
System checks nearest shops having the items

Order Tracking
Sees order status in real-time:
Order Placed
Shop Accepted
Shop Ready
Delivery Agent Assigned
Picked Up
Out for Delivery
Delivered

Can cancel order (only before shop accepts)



2. Shop Owner Flow

Registration & Approval
Shop owner registers (shop name, address, contact, GST if any)
Admin reviews and approves the shop

Login & Dashboard
Logs in as Shop Owner
Sees dashboard with today’s orders and low stock items

Inventory Management
Gets pre-filled list of products in their account
Updates quantity using + and – buttons
Marks items as Out of Stock if needed

Order Handling
Receives new order notification
Views order details (items, customer address, total)
Accepts or Rejects the order (with reason if rejected)
Prepares the order
Clicks "Mark as Ready" when items are packed

Order Completion
Hands over order to Delivery Agent when they arrive
Can view order history and earnings



3. Delivery Agent Flow

Registration & Approval
Delivery agent registers (Name, Phone, Vehicle details)
Admin approves the agent

Login
Logs in as Delivery Agent
Goes online (available for orders)

Order Pickup
Sees available orders in nearby area
Accepts order request
Goes to the shop
Clicks "Arrived at Shop"
Verifies items with shop owner
Clicks "Order Picked Up"

Delivery Process
Navigates to customer location (basic map or address shown)
Clicks "Out for Delivery"
Reaches customer
Delivers order and clicks "Delivered"
Can collect payment if Cash on Delivery

Earnings & History
Can see total earnings and completed deliveries



4. Admin Flow

Login
Admin logs in with special credentials

Dashboard
Overview: Total orders, active shops, active delivery agents, revenue

User Management
Approve or reject new Shop Owners
Approve or reject new Delivery Agents
View and manage all Customers, Shops, and Delivery Agents
Can block any user

Order Management
View all orders across the platform
Can cancel or modify orders in special cases

Shop & Inventory Oversight
Can view all shops and their inventory status
Can add or remove products from master product list

System Control
Manage categories and products in master list
View reports (daily orders, most selling items, etc.)
Handle disputes or complaints



Complete End-to-End Flow (Most Important)

Customer places order
System sends order to nearest 2–3 shops
Shop owner accepts the order
System sends order to nearby delivery agents
Delivery agent accepts the order
Delivery agent goes to shop → Picks up order
Delivery agent delivers to customer → Marks delivered
Shop owner gets payment (after commission deduction)
